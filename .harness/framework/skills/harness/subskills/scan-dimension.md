# Subskill: 扫描维度（通用模板）

## 任务

按指定维度扫描源代码，逐文件读取，输出违规清单。

## 输入

| 参数 | 必需 | 说明 |
|------|------|------|
| dimension | 是 | 维度名称（如"架构边界""编码约定"） |
| rule_sources | 是 | 规则来源列表（.harness/PROJECT.md 章节名 和/或 knowledge/ 文件路径） |
| files | 否 | 文件路径列表。未提供时，读取 .harness/PROJECT.md "仓库结构" 确定源代码目录 |

## 执行

### Todo 1 -- 提取检查规则
依次读取 rule_sources 中的每个来源，提取该维度的检查规则和已知例外

### Todo 2 -- 逐文件扫描
逐文件对照检查规则扫描，发现违规记录到输出

### Todo 3 -- 例外判定
规则来源中未声明例外时，所有偏离均视为违规；测试文件中的硬编码测试数据、条件编译调试块（#if DEBUG）不视为违规；但测试文件中硬编码的 API 密钥、凭证等安全敏感数据仍视为违规

## 输出格式

```
## {dimension} 扫描结果
违规数量：N
| # | 文件 | 行号 | 违规项 | 建议修复 |
|---|------|------|--------|---------|
未发现违规的检查项：...
```
