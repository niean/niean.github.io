以下为常见扫描维度参考，按项目需要选择和定制：

### 架构边界（参考）
- UI 层不直接调用底层 Service，须通过 Coordinator 或 Manager
- 全屏页面统一管理方式
- 网络请求设超时
- 异步操作非主线程，回调切回主线程更新 UI
- 大数据集合设上限

### 编码约定（参考）
- 常量集中管理，不散落各文件
- 保留字/特殊值处理
- 手势/导航约定
- 文档/注释格式

### 安全规范（参考）
- 密钥只存安全存储（如 Keychain），不硬编码
- 密钥日志脱敏
- 配置文件不入库
- 强制 HTTPS
- API 响应校验
- 用户数据本地存储

### 日志规范（参考）
- 统一日志框架（如 os.Logger），禁止 print()
- 日志格式规范
- 错误信息分层（用户提示 vs 开发日志）
- 禁止输出敏感信息

### 废弃代码（参考）
- 未使用的类型/函数/变量
- 未使用的文件
- 无效导入
- 过期注释
