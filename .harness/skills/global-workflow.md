# Global Workflow（全局工作流）

定位：迭代类 Skill 的执行守护，定义每次迭代类任务必须遵循的 6 阶段标准流程。

目的：防止 AI 遗忘任务执行步骤。AI 每次启动任务时，AGENTS.md 中的摘要必然可见；需要详细规则时读取本文件。

适用范围：遵循 Global Workflow 的 Skill。其他 Skill 有各自流程，不受本工作流约束。

---

## 6 阶段定义

| 阶段 | 名称 | GATE | 语义 |
|------|------|------|------|
| 1 | 任务分解 | - | 读取 AGENTS.md 约束，确认产品方向，识别任务类型，路由到具体 Skill |
| 2 | 意图识别 | - | 分析需求，按需读取知识库和产品文档，产出结构化 spec（JSON） |
| 3 | 意图确认 | `[GATE]` | spec 落盘，向用户输出完整摘要，等待确认；修正时重新输出摘要再次确认 |
| 4 | 任务实现(含验收) | `[GATE-ENTRY]` | 按 spec 执行实现，对照验收标准检查，不通过时就地修正 |
| 5 | 知识回填 | - | 按 AGENTS.md 知识回填规则回填 context/agents/，删除临时 spec |
| 6 | 任务总结 | `[GATE]` | 自动触发 Skill: 总结任务，输出报告 -> 用户确认 -> attempt_completion |

---

## 阶段详细规则

### 阶段 1: 任务分解

- 输入：用户请求
- 操作：读取 AGENTS.md，确认约束与产品方向，识别任务类型并路由到具体 Skill
- 输出：任务声明（任务类型 + 调度架构），启动阶段 2
- 约束：任务声明格式遵守 AGENTS.md "消息输出格式"

### 阶段 2: 意图识别

- 输入：用户请求 + AGENTS.md 约束
- 操作：分析需求，按需读取知识库和产品文档，产出结构化 spec
- 输出：spec（JSON 格式，含目标、影响范围、实现思路、验收标准）
- 检查点模板：`[阶段 2 意图识别] goal: ..., scope: N 文件, M 行为, K 验收标准`
- Skill 差异：迭代功能由 Analyst subagent 执行，迭代其它由 Orchestrator 直接执行

### 阶段 3: 意图确认 `[GATE]`

- 输入：阶段 2 产出的 spec
- 操作：
  1. spec 落盘到 `.harness/context/agents/agent-specs-${事项}.md`
  2. 向用户输出完整摘要（目标 + 影响范围 + 实现思路 + 验收标准）
  3. 使用 `ask_followup_question` 请求用户确认，立即结束当前回复
- 修正循环：用户修正时更新 spec，重新输出完整摘要，再次走 GATE 确认；用户修正 ≠ 用户确认
- 输出：用户确认后的最终 spec
- GATE 规则：每次输出完整摘要后必须立即结束回复，禁止在同一条回复中继续阶段 4

### 阶段 4: 任务实现(含验收) `[GATE-ENTRY]`

- 前置条件：用户已在上一条消息中明确确认 spec
- 输入：确认后的 spec
- 操作：按 spec 执行实现，完成后对照验收标准逐项检查，不通过时就地修正
- 输出：实现产物 + 验收结果
- 检查点模板：`[阶段 4 任务实现] 变更: file1(修改), file2(新增), ...; 验收: K项通过`
- Skill 差异：
  - 迭代功能：Phase 4 内含 Step 4.1 代码实现（Coder）+ Step 4.2 结果验收（Reviewer）
  - 迭代其它：Orchestrator 合并执行实现与验收

### 阶段 5: 知识回填

- 输入：阶段 4 的变更清单
- 操作：
  1. 按 AGENTS.md 知识回填规则回填 context/agents/（有变化才写，无变化也告知）
  2. `rm -f` 删除临时 spec
- 输出：回填结果摘要

### 阶段 6: 任务总结 `[GATE]`

- 输入：全流程检查点摘要
- 操作：自动触发 Skill: 总结任务（`.harness/skills/summarize-task.md`）
- 执行顺序：输出总结报告（通过 `ask_followup_question`）-> 用户确认收到 -> `attempt_completion`
- GATE 规则：总结报告输出后必须立即结束回复，禁止在同一条回复中调用 `attempt_completion`
- 总结报告与 attempt_completion 不得合并

## Skill 映射

| Global Workflow 阶段 | 迭代功能 Phase | 迭代其它 Phase |
|---------------------|---------------|---------------|
| 1 任务分解 | Phase 1 任务调度 | Phase 1 任务调度 |
| 2 意图识别 | Phase 2 意图理解 | Phase 2 意图理解 |
| 3 意图确认 | Phase 3 意图确认 | Phase 3 意图确认 |
| 4 任务实现(含验收) | Phase 4 任务实现(含验收) | Phase 4 任务实现 |
| 5 知识回填 | Phase 5 知识回填 | Phase 5 知识回填 |
| 6 任务总结 | Phase 6 任务总结 | Phase 6 任务总结 |

---

## 上下文管理

- 每个阶段完成后输出检查点摘要（不超过 10 行），后续只携带摘要
- 阶段 3 后仅保留 spec，不保留产品文档原文
- 阶段 4 只加载 spec + scope 内相关文件
- 上下文紧张时先压缩检查点摘要再继续
- 各 Skill 如有更具体的上下文管理要求，以 Skill 文件为准
